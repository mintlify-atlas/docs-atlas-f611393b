The Windsurf adapter writes a single `.windsurfrules` file at your project root. Windsurf's Cascade reads it on every session start and prepends it to the system prompt, so the model always knows to consult `.agent/` before acting. Like Cursor, Windsurf does not expose a native post-tool hook, so memory logging is instruction-driven by default — with the standalone-python conductor available as an automatic side channel.

## What gets installed

| File | Purpose |
|---|---|
| `.windsurfrules` | Project-root rules file read by Cascade on every session start |

## Supported harnesses reference

| Harness | Config file | Hook support |
|---|---|---|
| **Claude Code** | `CLAUDE.md` + `.claude/settings.json` | yes (PostToolUse, Stop) |
| **Cursor** | `.cursor/rules/*.mdc` | no (manual reflect calls) |
| **Windsurf** | `.windsurfrules` | no (manual reflect calls) |
| **OpenCode** | `AGENTS.md` + `opencode.json` | partial (permission rules) |
| **OpenClaw** | system-prompt include | varies by fork |
| **Hermes Agent** | `AGENTS.md` (agentskills.io compatible) | partial (own memory) |
| **Pi Coding Agent** | `AGENTS.md` + `.pi/skills/` | no (extension system) |
| **Standalone Python** | `run.py` (any LLM) | yes (full control) |

## Install steps

<Steps>
  <Step title="Run the installer">
    From your project root, run the adapter installer. The onboarding wizard launches automatically and populates `.agent/memory/personal/PREFERENCES.md`.

    <CodeGroup>

    ```bash macOS / Linux
    ./install.sh windsurf
    ```

    ```powershell Windows
    .\install.ps1 windsurf C:\path\to\your-project
    ```

    </CodeGroup>
  </Step>
  <Step title="Verify the rules file is active">
    Open Windsurf and ask Cascade:

    ```
    What's in your lessons file?
    ```

    Cascade should read `.agent/memory/semantic/LESSONS.md`. If it does, the rules file is being loaded.
  </Step>
</Steps>

## How Windsurf loads the rules file

Windsurf's Cascade reads `.windsurfrules` from the exact project root on every session start and prepends its contents to the system prompt. The file is read on every interaction — keep it under roughly 200 lines to avoid excessive token use on short tasks.

<Warning>
Windsurf does not search subdirectories for the rules file. It must be at the project root (the directory you open in Windsurf), not inside a subdirectory. If the agent is not following the instructions, confirm the file location first.
</Warning>

## Memory logging

Windsurf does not have a first-class post-tool hook. The `.windsurfrules` file instructs the model to call `memory_reflect.py` after significant actions, but enforcement depends on the model following the instruction.

**Manual (default):** Call reflect from the terminal whenever you want to log a session observation:

```bash
python3 .agent/tools/memory_reflect.py windsurf manual ok
```

**Automatic via side channel:** Run the standalone-python conductor in a parallel terminal session. It owns its own loop, logs every action, and runs the dream cycle on exit — giving Windsurf sessions the same memory-compounding behavior as Claude Code without requiring hook support.

```bash
python run.py "reflect on today's work"
```

<Tip>
If consistent, gap-free memory logging matters for your workflow, the standalone-python side channel is the more reliable option. See the [standalone-python guide](/harnesses/standalone-python) for full setup.
</Tip>

## Customizing the rules file

Edit `.windsurfrules` directly to add or change instructions injected into every Cascade session. The file is plain Markdown — no special frontmatter is required.

Keep the file focused. Windsurf loads it on every session regardless of task, so content that is only relevant to a narrow scenario adds overhead without benefit. Project-specific overrides are better placed in a separate instruction block at the bottom of the file, separated by a heading.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Cascade ignores .windsurfrules">
    Confirm the file is at the exact project root — the same directory you opened in Windsurf. Windsurf does not walk up the directory tree or scan subdirectories. If you have a monorepo, each sub-project that needs the rules requires its own copy of the file.
  </Accordion>
  <Accordion title="Agent does not call memory_reflect.py">
    This is expected: Windsurf has no hook enforcement. Either remind the model explicitly in your prompt, or use the standalone-python side channel for automatic logging.
  </Accordion>
  <Accordion title="Rules file causes context overflow on short tasks">
    Trim `.windsurfrules` below 200 lines. Move rarely-needed detail into skill `SKILL.md` files under `.agent/skills/` — those load only when their triggers match, keeping the always-on context budget small.
  </Accordion>
</AccordionGroup>
