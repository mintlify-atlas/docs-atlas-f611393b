Drop the `.agent/` directory into any project and your AI gets a working portable brain that carries its knowledge when you change tools.

## The core concept: harness-agnostic portable brain

Most AI coding assistants store their context inside the harness — which means switching from Claude Code to Cursor, or from Cursor to Windsurf, means starting from scratch. agentic-stack separates the brain from the harness. The `.agent/` folder is self-contained: it holds everything the AI needs to remember, reason, and improve. The harness only contributes the glue — a config file or a rules file — that tells it where to look.

The principle from `architecture.md` captures it directly: *the harness is dumb, the knowledge is in files.*

## Repo layout

```

├── AGENTS.md                   # the map
├── harness/                    # conductor + hooks (standalone path)
├── memory/                     # working / episodic / semantic / personal
│   ├── auto_dream.py           # staging-only dream cycle
│   ├── cluster.py              # content clustering + pattern extraction
│   ├── promote.py              # stage candidates
│   ├── validate.py             # heuristic prefilter (length + exact duplicate)

    ├── list_candidates.py
    ├── graduate.py
    ├── reject.py
```

## Supported harnesses

agentic-stack works with all major AI coding environments through adapter shims that require no changes to the `.agent/` folder itself.

| Harness | Config file it reads | Hook support |
|---|---|---|
| **Claude Code** | `CLAUDE.md` + `.claude/settings.json` | yes (PostToolUse, Stop) |
| **Cursor** | `.cursor/rules/*.mdc` | no (manual reflect calls) |
| **Windsurf** | `.windsurfrules` | no (manual reflect calls) |
| **OpenCode** | `AGENTS.md` + `opencode.json` | partial (permission rules) |
| **OpenClaw** | system-prompt include | varies by fork |
| **Hermes Agent** | `AGENTS.md` (agentskills.io compatible) | partial (own memory) |
| **Pi Coding Agent** | `AGENTS.md` + `.pi/skills/` | no (extension system) |
| **Standalone Python** | `run.py` (any LLM) | yes (full control) |

## Key features

**Memory layers.** Four layers — `working/`, `episodic/`, `semantic/`, `personal/` — each with its own retention policy. Query-aware retrieval ranks entries by salience and relevance. Episodic events compress nightly into reviewable candidate lessons.

**Skills.** Progressive disclosure keeps context lean. A lightweight `_index.md` and `_manifest.jsonl` are always loaded; a full `SKILL.md` only enters context when the task matches its triggers. Every skill ships with a self-rewrite hook that fires after repeated failures.

**Protocols.** `permissions.md` defines what is allowed, what requires approval, and what is never allowed. A pre-tool-call hook enforces it. `tool_schemas/` provides typed interfaces for every external tool. `delegation.md` governs sub-agent handoff.

**Review protocol.** `auto_dream.py` runs nightly and stages candidate lessons mechanically — it clusters patterns, runs a heuristic prefilter, and writes a review queue. It never reasons, never promotes, and never commits. The host agent reviews staged candidates via CLI tools (`graduate.py`, `reject.py`, `reopen.py`) and must supply a rationale to graduate a lesson. No unattended reasoning.

**Dream cycle.** The nightly cycle clusters episodic events, extracts recurring patterns, prefilters obvious junk, decays old entries, and writes `REVIEW_QUEUE.md`. The next session opens with a clear picture of what needs review.
