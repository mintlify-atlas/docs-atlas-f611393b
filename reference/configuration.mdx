The `.agent/` folder contains several files that shape how the AI behaves in your project. Some are written by you, some are updated by the agent during a session, and some are auto-generated by background processes. This page lists all of them, explains their purpose, and tells you which ones are safe to edit by hand.

## Configuration file overview

| File | Path | Edited by |
|---|---|---|
| `PREFERENCES.md` | `.agent/memory/personal/PREFERENCES.md` | Human |
| `WORKSPACE.md` | `.agent/memory/working/WORKSPACE.md` | Agent |
| `REVIEW_QUEUE.md` | `.agent/memory/working/REVIEW_QUEUE.md` | Auto-generated |
| `DECISIONS.md` | `.agent/memory/semantic/DECISIONS.md` | Human + Agent |
| `LESSONS.md` | `.agent/memory/semantic/LESSONS.md` | Auto-generated (do not edit) |
| `lessons.jsonl` | `.agent/memory/semantic/lessons.jsonl` | Auto-generated (do not edit) |
| `.features.json` | `.agent/memory/.features.json` | Human |

| `_index.md` | `.agent/skills/_index.md` | Human + Agent |
| `_manifest.jsonl` | `.agent/skills/_manifest.jsonl` | Human + Agent |

## Memory files

### PREFERENCES.md

**Path:** `.agent/memory/personal/PREFERENCES.md`
**Edited by:** Human

The first file your AI reads at the start of every session. Contains your name, preferred languages, explanation style, test strategy, commit style, review depth, and any project-specific constraints. Written by the onboarding wizard; edit it any time to refine your conventions.

```markdown
## Code style
- Language(s): TypeScript, Python
- Explanations: concise

## Workflow
- Test strategy: test-after
- Commit style: conventional commits

## Communication
- Review depth: critical issues only
- Tone: direct, skip pleasantries
- Surface tradeoffs: always

## Constraints
- _(edit me: primary stack, deployment targets, off-limits patterns)_
```

### WORKSPACE.md

**Path:** `.agent/memory/working/WORKSPACE.md`
**Edited by:** Agent

The agent's current task state. Updated as the agent works through a task and archived at completion. Reading this at session start lets the agent resume interrupted work without losing context. Do not delete active task state — the agent archives it automatically when the task is done.

### REVIEW_QUEUE.md

**Path:** `.agent/memory/working/REVIEW_QUEUE.md`
**Edited by:** Auto-generated by `auto_dream.py`

A summary of how many candidate lessons are waiting for review and how long the oldest one has been staged. The agent checks this at session start; if pending candidates exceed 10 or the oldest is more than 7 days old, the agent handles reviews before starting new work.

### DECISIONS.md

**Path:** `.agent/memory/semantic/DECISIONS.md`
**Edited by:** Human and Agent

A log of past architectural choices. The agent reads this before making decisions it has been corrected on before, and may append new decisions here during a session.

### LESSONS.md

**Path:** `.agent/memory/semantic/LESSONS.md`
**Edited by:** Auto-generated — do not hand-edit

<Warning>
  `LESSONS.md` is rendered from `lessons.jsonl` by `render_lessons.py`. Any
  content you write directly into this file below the sentinel line will be
  overwritten the next time `graduate.py` runs. To add a lesson, use
  `graduate.py` with a rationale. Hand-curated content placed above the
  sentinel is preserved across renders.
</Warning>

A distilled set of patterns the agent has learned and the host agent has reviewed and accepted. Loaded at session start for query-relevant retrieval.

### lessons.jsonl

**Path:** `.agent/memory/semantic/lessons.jsonl`
**Edited by:** Auto-generated — do not hand-edit

<Warning>
  `lessons.jsonl` is the source of truth for graduated lessons. Editing it
  directly will corrupt the lesson lifecycle. Use `graduate.py` and
  `reject.py` to add and remove lessons.
</Warning>

Each line is a JSON object representing one accepted lesson, including its claim, conditions, evidence IDs, rationale, reviewer, and status (`accepted` or `provisional`). `LESSONS.md` is rendered from this file.

## Feature toggles

### .features.json

**Path:** `.agent/memory/.features.json`
**Edited by:** Human

Controls opt-in beta features. Written by the onboarding wizard and safe to edit directly at any time.

```json
{
  "memory_search_fts": {
    "enabled": false,
    "beta": true
  }
}
```

Set `"enabled": true` to activate full-text memory search. All features default to `false` — the file follows an explicit opt-in model.

## Protocol files

### permissions.md

**Path:** `.agent/protocols/permissions.md`
**Edited by:** Human only

<Warning>
  The agent is unconditionally blocked from modifying `permissions.md`. Only
  humans edit this file. Changing it requires opening it in an editor directly
  — it cannot be modified through any agent tool call.
</Warning>

Defines three permission tiers enforced by the `pre_tool_call` hook before every tool invocation: always-allowed actions, actions that require human approval, and actions that are never allowed. Also lists the five approved external domains for HTTP requests. See [Permissions, delegation, and tool schemas](/reference/protocols) for the full rule set.

## Skill files

### _index.md

**Path:** `.agent/skills/_index.md`
**Edited by:** Human and Agent

The skill registry — a human-readable index of all installed skills with their trigger phrases and brief descriptions. The agent reads this first for discovery; full `SKILL.md` files are only loaded when a trigger phrase matches the current task.

### _manifest.jsonl

**Path:** `.agent/skills/_manifest.jsonl`
**Edited by:** Human and Agent

Machine-readable skill metadata. Each line is a JSON object with the skill's name, triggers, constraints, and preconditions. Used by `skill_loader.py` for progressive disclosure — the manifest is always loaded; the full skill content is deferred until needed.

## Environment variables

Copy `.env.example` to `.env` and fill in your provider credentials. The standalone-python harness reads these directly; other harnesses pick them up through the shell environment.

```bash
# Pick one provider
ANTHROPIC_API_KEY=sk-ant-...
# OPENAI_API_KEY=sk-...

# Optional overrides
AGENT_PROVIDER=anthropic
AGENT_MODEL=claude-sonnet-4-5
AGENT_MAX_CONTEXT=128000
```

| Variable | Purpose |
|---|---|
| `ANTHROPIC_API_KEY` | Anthropic provider credentials. |
| `OPENAI_API_KEY` | OpenAI provider credentials (alternative). |
| `AGENT_PROVIDER` | Which provider to use (`anthropic` or `openai`). |
| `AGENT_MODEL` | Model name override. |
| `AGENT_MAX_CONTEXT` | Context window size in tokens. |
