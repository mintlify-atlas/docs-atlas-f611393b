agentic-stack installs with a single Homebrew tap on macOS and Linux, or a PowerShell script on Windows. After installation, a short onboarding wizard writes your preferences to `.agent/memory/personal/PREFERENCES.md` — the first file your AI reads at the start of every session.

<Steps>
  <Step title="Install on macOS or Linux">
    Tap the repository and install the package. Both lines are required on first install.

    ```bash
    brew tap codejunkie99/agentic-stack https://github.com/codejunkie99/agentic-stack
    brew install agentic-stack
    ```

    To upgrade an existing install:

    ```bash
    brew update && brew upgrade agentic-stack
    ```
  </Step>

  <Step title="Install on Windows">
    On Windows, clone the repository and run the native PowerShell installer directly.

    ```powershell
    git clone https://github.com/codejunkie99/agentic-stack.git
    cd agentic-stack
    .\install.ps1 claude-code C:\path\to\your-project
    ```

    Replace `claude-code` with your harness and the path with your project directory. The PowerShell installer accepts the same flags as the shell script (`-Yes`, `-Reconfigure`, `-Force`).

    <Note>
      The PowerShell installer (`install.ps1`) runs natively. You can also use `install.sh` under Git Bash or WSL.
    </Note>
  </Step>

  <Step title="Alternative: clone-based install">
    If you prefer not to use Homebrew, clone the repository and run `install.sh` directly.

    ```bash
    git clone https://github.com/codejunkie99/agentic-stack.git
    cd agentic-stack && ./install.sh claude-code
    # adapters: claude-code | cursor | windsurf | opencode | openclaw | hermes | pi | standalone-python
    ```

    You can pass a target directory as the second argument to install into a project other than the current directory:

    ```bash
    ./install.sh claude-code /path/to/your-project
    ```
  </Step>

  <Step title="Run the onboarding wizard">
    After the adapter is installed, the wizard runs automatically. It populates `.agent/memory/personal/PREFERENCES.md` and writes feature toggles to `.agent/memory/.features.json`.

    The wizard asks six questions (press Enter to skip any and accept the default):

    | Question | Default |
    |---|---|
    | What should I call you? | *(skip)* |
    | Primary language(s)? | `unspecified` |
    | Explanation style? | `concise` |
    | Test strategy? | `test-after` |
    | Commit message style? | `conventional commits` |
    | Code review depth? | `critical issues only` |

    There is also one optional feature step:

    | Feature | Default |
    |---|---|
    | Enable FTS memory search `[BETA]` | `no` |

    **Wizard flags**

    <CodeGroup>
      ```bash macOS / Linux
      # accept all defaults without prompting (CI-safe)
      agentic-stack claude-code --yes

      # re-run the wizard on an existing project
      agentic-stack claude-code --reconfigure
      ```

      ```powershell Windows
      # accept all defaults without prompting (CI-safe)
      .\install.ps1 claude-code -Yes

      # re-run the wizard on an existing project
      .\install.ps1 claude-code -Reconfigure
      ```
    </CodeGroup>

    <Tip>
      You can edit `.agent/memory/personal/PREFERENCES.md` at any time to refine your conventions. Your AI re-reads it at the start of every session.
    </Tip>
  </Step>

  <Step title="(Optional) Set up the nightly dream cycle">
    The dream cycle clusters episodic events into candidate lessons and updates your review queue. Schedule it with cron to run automatically each night.

    ```bash
    crontab -e
    ```

    Add this line, replacing the path with your actual project path:

    ```bash
    0 3 * * * python3 /path/to/project/.agent/memory/auto_dream.py >> /path/to/project/.agent/memory/dream.log 2>&1
    ```

    `auto_dream.py` resolves paths absolutely and performs only mechanical file operations — clustering, staging, prefiltering, and decay. It makes no git commits, no network calls, and no subjective judgments. Safe to run unattended.
  </Step>
</Steps>

## Verify the wiring

Once installed, confirm that the memory files are being read correctly by checking the token budget:

```bash
python3 .agent/tools/budget_tracker.py "commit and push"
# outputs: tokens_used, chars, budget, headroom
```

If `tokens_used` is `0`, the memory files are not being loaded — check that the paths in your harness config point to the correct `.agent/` location.

## What was just installed

The installer copied the `.agent/` brain into your project (if it was not already there) and dropped a harness-specific config file. For Claude Code, that is `CLAUDE.md` and `.claude/settings.json`. For other harnesses, see the table below:

| Harness | Files added to your project |
|---|---|
| `claude-code` | `CLAUDE.md`, `.claude/settings.json` |
| `cursor` | `.cursor/rules/agentic-stack.mdc` |
| `windsurf` | `.windsurfrules` |
| `opencode` | `AGENTS.md`, `opencode.json` |
| `openclaw` | `.openclaw-system.md` |
| `hermes` | `AGENTS.md` |
| `pi` | `AGENTS.md`, `.pi/skills` symlink |
| `standalone-python` | `run.py` |

<Info>
  The `.agent/` folder is the same across all harnesses. Only the glue file changes. You can install multiple adapters into the same project without duplicating the brain.
</Info>
